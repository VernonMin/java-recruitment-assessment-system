# 产品需求文档

## 1. 产品概述

### 1.1 产品名称

Java 招聘测评系统

归属公司：昱易昇（成都）科技有限公司

### 1.2 产品定位

一个面向企业招聘 Java 开发岗位的在线测评平台，用于远程筛选候选人。

当前前端形态：

- 一套 `求职者端`，给候选人使用
- 一套 `企业端`，给面试官、招聘专员、管理员使用
- 一套独立 `登录页`，未登录时不显示业务导航

### 1.3 产品目标

- 支持 Java 开发候选人的线上初筛测评
- 为招聘专员和面试官提供题库、组卷、发布、评估的完整流程
- 为候选人提供稳定、清晰的桌面端答题体验
- 提供基础防作弊能力，满足远程招聘场景

### 1.4 非目标

- MVP 不做在线代码执行沙箱
- MVP 不做实时音视频监考
- MVP 不做 AI 作弊识别
- MVP 不做外部 SSO 集成

## 2. 用户角色

### 2.1 候选人

主要职责：

- 登录求职者端并进入分配给自己的测评
- 完成客观题与主观题
- 在规定时间内提交答案
- 在允许的情况下查看结果

### 2.2 面试官

主要职责：

- 创建和维护招聘题目
- 组装测评模板
- 评审主观题答案
- 填写评分意见与评价
- 在企业端使用题库和评估模块

### 2.3 招聘专员

主要职责：

- 创建招聘试题
- 分配候选人
- 跟踪邀请、开始、完成状态
- 汇总结果并形成招聘建议
- 在企业端推进招聘筛选流程

### 2.4 管理员

主要职责：

- 管理账号与角色
- 管理系统配置
- 审计测评过程
- 查看监控事件
- 统一管理企业端入口与权限

## 3. 核心场景

### 3.1 候选人参加测评

1. 候选人使用账号密码登录
2. 候选人看到分配给自己的招聘测评
3. 候选人进入测评并授权必要权限
4. 候选人完成答题并提交
5. 系统自动完成客观题评分
6. 面试官评审主观题
7. 招聘专员或管理员决定是否对候选人开放结果

### 3.2 面试官准备测评

1. 面试官创建招聘题目
2. 面试官将题目组装为测评模板
3. 招聘专员基于模板创建招聘试题
4. 招聘专员分配候选人与测评规则
5. 面试官查看提交并进行评估

### 3.3 招聘专员查看筛选结果

1. 招聘专员跟踪候选人完成进度
2. 招聘专员查看评分、评语和异常监控事件
3. 招聘专员填写推荐结论，如待定或淘汰
4. 招聘专员导出结果汇总

## 4. 功能需求

### 4.1 登录与权限

要求：

- 支持账号密码登录
- 支持安全会话管理
- 支持基于角色的权限控制
- 登录后按端自动跳转：候选人进入求职者端，企业角色进入企业端
- 未登录时只显示独立登录页
- 支持账号禁用与锁定
- 支持招聘专员或管理员重置密码

优先级：P0

### 4.2 用户管理

要求：

- 支持创建候选人、面试官、招聘专员、管理员账号
- 支持批量导入账号
- 支持按关键词、角色、状态检索账号
- 检索结果支持服务端分页查询
- 支持编辑用户资料与状态
- 支持重置密码
- 支持删除未被引用的账号
- 支持一个用户绑定多个角色
- 支持跟踪候选人邀请状态
- 支持单个或批量将候选人账号分配到具体招聘试题

优先级：P0

### 4.3 题库管理

支持题型：

- 单选题
- 多选题
- 判断题
- 填空题
- 简答题
- 场景分析题

要求：

- 创建、编辑、发布、归档题目
- 支持按关键词、题型、状态检索题目
- 题目检索结果支持服务端分页查询
- 支持删除未被引用的题目
- 为客观题配置标准答案
- 配置分值、难度、标签
- 为面试官保留解析说明

优先级：P0

### 4.4 测评模板管理

要求：

- 从题库中选择题目组成测评模板
- 调整题目顺序与分值
- 支持模板列表检索与分页浏览
- 支持修改已有模板
- 支持模板复用

优先级：P0

### 4.5 招聘试题管理

要求：

- 基于测评模板创建招聘试题
- 支持修改已有试题
- 支持按关键词、状态检索招聘试题
- 招聘试题列表支持服务端分页查询
- 配置开始时间、结束时间、时长
- 分配候选人
- 配置是否要求摄像头
- 配置是否要求全屏
- 支持发布与归档

优先级：P0

### 4.6 候选人答题体验

要求：

- 显示剩余时间
- 自动保存答案
- 支持手动提交
- 超时自动提交
- 试题结束后默认禁止进入

优先级：P0

### 4.7 提交详情与评估

要求：

- 支持按关键词、状态、招聘试题检索提交记录
- 提交记录列表支持服务端分页查询
- 支持从提交列表直接进入详情页
- 支持查看逐题作答、客观题得分、主观题得分和评语
- 支持在详情基础上进入人工评估

优先级：P0

### 4.7 自动评分

要求：

- 客观题自动评分
- 支持精确匹配与集合匹配
- 自动汇总总分
- 主观题标记为待人工评估

优先级：P0

### 4.8 人工评估

要求：

- 面试官查看主观题答案
- 支持调用 AI 生成建议分数与评语
- AI 建议结果必须由人工确认后才能写入最终成绩
- 面试官填写分数与评语
- 支持复评
- 系统保留评估历史

优先级：P0

### 4.9 结果管理

要求：

- 候选人按权限查看结果
- 面试官查看详细提交与评分
- 招聘专员查看汇总结果与推荐结论
- 招聘专员和管理员支持导出报表

优先级：P1

### 4.10 基础监控

要求：

- 记录摄像头授权结果
- 在启用时上传周期性抓拍
- 记录页面失焦与退出全屏
- 记录断网与恢复联网
- 为每次提交生成基础风险等级

优先级：P1

## 5. 非功能需求

### 5.1 安全性

- 密码必须使用 `bcrypt` 或 `argon2` 哈希存储
- 会话 Cookie 必须为 `HttpOnly` 和 `Secure`
- API 必须校验角色权限
- 敏感操作必须有审计记录

### 5.2 性能

- 常规并发下登录和页面加载应保持可用
- 客观题评分应在可接受时延内异步完成

### 5.3 可靠性

- 自动保存要尽量降低答案丢失风险
- 提交接口应尽量保证幂等
- 监控事件不能阻塞答题提交流程

### 5.4 隐私

- 摄像头使用必须经过候选人明确授权
- 抓拍数据保留周期必须可配置
- 监控数据只能由授权角色访问

## 6. MVP 范围

包含：

- 账号密码登录
- 候选人、面试官、招聘专员、管理员角色
- Java 招聘题库
- 测评模板管理
- 招聘试题创建与发布
- 在线答题
- 客观题自动评分
- 主观题人工评估
- 结果与推荐结论查看
- 抓拍与页面行为事件记录

不包含：

- 编程题执行
- 实时视频监考
- 语音录制
- AI 行为分析
- 公开自助报名

## 7. 成功指标

- 招聘专员可以独立完成试题创建和发布
- 候选人可以独立完成测评，不依赖人工干预
- 客观题可以自动评分
- 主观题评估流程完整且可追溯
- 每次提交的监控事件可查询

## 8. 风险

- 候选人可能拒绝摄像头权限
- 不同浏览器的全屏行为存在差异
- 如果题目设计过于通用，会削弱 Java 岗位筛选效果
