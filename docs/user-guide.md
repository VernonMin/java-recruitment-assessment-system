# 系统使用说明

## 1. 说明

本文档说明 MVP 阶段各角色如何使用系统。
系统归属：昱易昇（成都）科技有限公司。

支持角色：

- 候选人
- 面试官
- 招聘专员
- 管理员

前端入口说明：

- `求职者端`：只给候选人使用
- `企业端`：给面试官、招聘专员、管理员共用
- 登录页为独立页面，未登录时不显示业务导航

当前管理员默认账号：

- `admin / admin`：管理员，进入企业端

## 2. 候选人使用说明

### 2.1 登录

1. 打开系统登录页
2. 输入账号和密码
3. 点击“登录”
4. 登录成功后自动进入求职者端

说明：

- 前端默认通过当前站点同源 `/api` 访问后端
- 正式环境下候选人不需要手工填写后端服务地址
- 这样可以避免无痕模式下第三方 Cookie 被拦截导致刷新后掉登录

### 2.2 退出登录

1. 点击页面右上角“退出登录”
2. 系统清理当前会话
3. 页面自动回到登录页

### 2.3 开始测评

1. 打开已分配测评列表
2. 选择一个可参加的测评
3. 点击“进入答题”
4. 系统立即创建一条 `进行中` 的答题记录，并开始计时
5. 页面展示剩余时间、摄像头状态、全屏状态、网络状态
6. 如果任务要求摄像头，浏览器会申请摄像头权限
7. 如果任务要求全屏，系统会尝试进入全屏

### 2.4 答题

答题行为说明：

- 单选、多选、判断、填空题当前统一使用文本输入
- 简答题使用文本输入
- 场景分析题支持较长文本输入
- 前端会把当前答案保存到本地草稿
- 刷新页面后，系统会尝试恢复当前进行中的答题记录和本地草稿

### 2.5 提交

1. 点击“提交”
2. 系统根据当前进行中的答题记录完成交卷
3. 等待提交成功提示
4. 如果剩余时间归零，系统会自动交卷
5. 自动交卷后会生成一条超时留痕事件

### 2.6 查看结果

1. 进入已完成测评列表
2. 打开某次测评记录
3. 查看客观题得分、主观题得分、总分
4. 企业端还可以查看该次提交的异常事件和抓拍截图

## 3. 面试官使用说明

### 3.1 管理题目

1. 登录企业端
2. 进入题库管理
3. 手工创建题目
4. 填写题型、题干、答案、分值、解析
5. 保存为草稿或发布

当前版本说明：

- 题型支持单选、多选、判断、填空、简答、场景分析
- 已支持按关键词、题型、状态检索题目
- 已支持修改题目与删除未被引用的题目
- 选择题选项使用逐行录入，格式为 `A|选项内容`

### 3.2 创建试卷模板

1. 进入试卷模板管理
2. 创建新模板
3. 可以手动选题，也可以先填写题型分值、目标总分和难度范围，再点击“生成试卷”
4. 系统会从已发布题目中自动生成一版试卷，并填入已选题目列表
5. 人工继续调整题目分组、顺序与模板内分值
6. 保存模板
7. 如需修改，可在模板列表中打开“编辑模板”

### 3.3 评估主观题

1. 登录企业端
2. 进入“答卷详情”
3. 按关键词、提交状态、审核状态、笔试任务筛选提交记录
4. 提交记录默认按提交时间倒序排列，最新提交优先
5. 点击某条记录的“查看详情”，系统会打开独立答卷详情弹窗
6. 在弹窗中查看逐题答案、风险留痕、抓拍截图与当前分数
7. 如需处理主观题，点击“进入人工/AI评分复核”
8. 如已启用 AI 建议判分，可先点击“AI 生成建议”
9. 查看每道主观题的 AI 建议分数与建议评语
10. 按需点击“采纳 AI 建议”，或人工自行修订最终分数与最终评语
11. 点击“确认并提交最终成绩”

### 3.4 查看结果

1. 打开“答卷详情”
2. 使用筛选栏检索目标提交记录
3. 点击“查看详情”打开答卷详情弹窗
4. 查看总分、客观题分数、主观题分数、审核状态、风险等级和异常事件
5. 查看抓拍截图时间线
6. 如有需要可删除当前答卷详情，系统会同步清理对应作答、评分和监控留痕
7. 如有需要再进入人工/AI 评分复核

## 4. 招聘专员使用说明

### 4.1 管理候选人

1. 使用管理员账号或招聘专员账号进入企业端
2. 管理员在“用户管理”中创建求职者账号
3. 管理员或招聘专员在“用户管理”中，通过搜索候选人并勾选多选列表，将候选人分配到笔试任务
4. 跟踪邀请状态与完成状态

### 4.2 发布笔试任务

1. 登录企业端
2. 进入笔试任务管理
3. 基于试卷模板创建笔试任务
4. 设置测评时间窗口与时长
5. 配置是否要求摄像头或全屏
6. 发布试题

### 4.3 查看推荐结论

1. 打开“答卷详情”
2. 按审核状态筛选待处理或已审核记录
3. 查看分数、评语与异常监控事件
4. 进入人工/AI 评分复核确认最终成绩与推荐结论
5. 导出汇总结果

## 5. 管理员使用说明

### 5.1 管理用户

1. 登录企业端
2. 进入用户管理
3. 创建候选人、面试官、招聘专员、管理员账号
4. 为新账号设置初始密码
5. 查看当前账号列表和角色

当前版本说明：

- 已支持创建账号
- 已支持批量导入账号
- 已支持按关键词、角色、状态检索账号
- 已支持分页浏览用户检索结果
- 已支持编辑账号、禁用账号、重置密码
- 已支持删除未被引用的账号
- 账号状态只用于控制登录权限：`正常`、`锁定`、`禁用`
- 候选人业务进度应独立使用流程状态表达，例如：`待作答`、`已作答`、`评估中`、`流程结束`、`已归档`
- 已参与过笔试流程的候选人，不建议删除账号；更合理的做法是保留账号和历史记录，并在流程结束后转为“流程结束”或“已归档”
- 已支持单个或批量把候选人分配到笔试任务
- 已支持在人工评估页调用 DeepSeek 兼容接口生成主观题建议分数

### 5.2 审计试题

1. 进入笔试任务管理后台
2. 先通过检索栏筛选目标任务，再用分页浏览结果
3. 使用“新增任务”按钮打开弹窗创建新任务
4. 使用任务卡片中的“编辑任务”或“修改任务”按钮打开弹窗更新已有任务
5. 查看任务状态与提交数量
6. 如有需要批量分配候选人
7. 查看异常监控事件
8. 导出报表

## 6. 监控与权限

### 6.1 摄像头

如果某场测评要求摄像头权限：

- 浏览器会向候选人发起授权请求
- 前端会按 60 秒周期抓拍
- 图片会上传保存，用于事后审查
- 如果候选人拒绝摄像头授权，会直接标记为高风险并记录原因

MVP 阶段这里不是实时视频监考。

### 6.2 全屏与页面行为

如果启用了相关规则，系统会记录：

- 退出全屏
- 页面失焦
- 网络断开
- 超时自动交卷

其中：

- 退出全屏：直接标记为高风险，并记录“候选人退出全屏作答”
- 拒绝摄像头：直接标记为高风险，并记录“候选人拒绝开启摄像头”
- 页面失焦、网络断开、超时自动交卷：按累计风险分参与计算

企业端在答卷详情中可以直接查看风险等级和高风险原因。

## 7. 常见问题

### 7.1 无法登录

请检查：

- 账号和密码是否正确
- 账号是否被禁用或锁定
- 浏览器是否允许 Cookie

### 7.2 摄像头授权失败

请检查：

- 浏览器是否允许摄像头权限
- 操作系统是否允许浏览器访问摄像头
- 是否有其他程序占用了摄像头

### 7.3 提交失败

请检查：

- 网络是否稳定
- 登录会话是否过期
- 测评时间是否已经结束

### 7.4 为什么题库里没有题

请检查：

- 是否已使用管理员或面试官账号登录

### 7.5 AI 建议判分为什么不能用

请检查：

- Worker 是否已配置 `AI_REVIEW_ENABLED=true`
- 是否已配置 `AI_REVIEW_API_KEY`
- `AI_REVIEW_BASE_URL` 与 `AI_REVIEW_MODEL` 是否正确
- 当前登录账号是否为面试官、招聘专员或管理员
- 是否已经手工创建题目
- 是否是首次部署，数据库中还没有正式题库数据

## 8. 浏览器建议

推荐使用：

- 最新版 Chrome
- 最新版 Edge

不建议在较旧浏览器中参加测评。
